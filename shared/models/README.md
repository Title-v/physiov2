# PhysioAI Shared Models

Browser-loadable model artifacts live here.

- `motion-tcn/manifest.json` describes the active AI-primary motion classifier.
- `motion-tcn/model.json` and weight shards are generated by `npm run train:tcn` or prepared by `npm run convert:tcn`.

Every manifest must include `landmarkSchemaId`, `modelInputLandmarks`,
`primaryRequiredLandmarks`, `stabilizerRequiredLandmarks`, `jointNames`,
`inputShape`, `phases`, `qualities`, and `approved`.

These files are served through `/shared/models/*` by the app's public symlink.
Keep trained model artifacts out of the frontend runtime bundle; they should
load lazily. If the manifest is missing, unapproved, or schema-incompatible,
runtime practice falls back to deterministic scoring.
